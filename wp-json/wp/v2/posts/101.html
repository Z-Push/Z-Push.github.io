Hi all,<\/p>\n

after 1,5 years of development we are happy to announce Z-Push 2.0 final.<\/p>\n

Some of the new features:
\n– supports ActiveSync version 14, allowing full HTML email synchronization on all capable devices.
\n– possibility to set\/unset your out-of-office settings on your mobile (not all devices support this feature)
\n– “z-push-top” tool which shows top-like informations about currently connected devices and their activity
\n– a real real-time notification mechanism for the Zarafa Backend (requires ZCP 7.0.6 or higher)
\n– performs semantic data checks before sending data to a mobile
\n– identifies and fixes or ignores broken items
\n– bi-directional and inter process loop detection
\n– server side search in emails which are not on the phone
\n– meeting organizer sees the status of attendees (accepted\/declined\/tentatively) on the phone
\n– notes synchronization (only available for some clients e.g. Touchdown on Android)
\n– heartbeat and partial synchronization support
\n– support of recurring tasks (Mantis #125)
\n– global synchronization of the Public Folder (Mantis #198)
\n– improved logging (loglevels, logs on user level possible)
\n– introducing the “combined” backend<\/p>\n

There is a lot of new code regarding the ActiveSync 14 support. Many synchronization mechanisms have changed.
\nWe also switched our Bugtracker. From this version on all tickets will be found at\u00a0http:\/\/jira.zarafa.com\/browse\/ZP<\/a>.<\/p>\n

We would like to thank everybody who contributed with testing and reporting of issues and ideas
\nespecially Jan aka deHoeninger, Vincent aka liverpoolfcfan, Erwin aka ebogaard,
\nUwe aka uwe347, Niels aka nprzybilla, Paul aka pfloor and Bert.
\nWe would also like to thank dw2412 \u2013 Andreas Brodowski \u2013 for his work on the AS12 branch. A few code blocks of his branch made it into the release.
\nApologies and a big thank you if we have forgotten someone.<\/p>\n

An updated version of the Mobile Device Management Plugin for the Zarafa Webaccess,
\nwhich supports Z-Push 2, was also released.
\nIt has a new feature which allows resynchronisation of the mobile devices.
\nPlease check the Community Hub for more information:
\nhttp:\/\/z-push.sf.net\/mdmplugin<\/a><\/p>\n

There is no upgrade path for Z-Push 1.x versions, but as this version implements
\na fully automatic resynchronizing of devices in the case states are missing or faulty,
\nyou should upgrade _without_ maintaining the existing states.<\/p>\n

The state directory is by default \/var\/lib\/z-push, the log directory \/var\/log\/z-push.
\nPlease create these directories and make them writeable for apache or
\nchange the configuration parameters to your needs.<\/p>\n

Be aware that if you are using LOGLEVEL_DEBUG and LOGLEVEL_WBXML Z-Push will be quite talkative, so it is advisable to use log-rotate on the log file.<\/p>\n

The complete changelog is available at:\u00a0http:\/\/zarafa-deutschland.de\/z-push-download\/final\/2.0\/z-push-2.0-1346.txt<\/a>.<\/p>\n

The version is available at:\u00a0http:\/\/z-push.sourceforge.net\/download<\/a>.<\/p>\n

Your feedback is highly appreciated.<\/p>\n

Kind regards,
\nManfred
