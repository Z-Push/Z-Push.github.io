Hi all,<\/p>\n

we have just released Z-Push 2.3.1 final (tag 2.3.1, commit 5b6688deeade1aad2fd9403914a08abdecbe3808).<\/p>\n

This is the first maintenance release of 2.3 containing a few important fixes related to Android calendaring, packages on RHEL 6 & 7, move operations and diff backends in general.
\nThe SQL state machine should now work with PostgreSQL, thanks to Ralf for finding the issue and fixing it.
\nZ-Push now also checks for non-standard characters in the SQL database name. Thanks to Dirk for reporting this.
\nThe issue that in some cases the iOS devices would loop forever and not sync should now also be fixed. Thanks to Mathias for reporting & testing.
\nFor more details and other changes, the full change log is available here<\/a>.<\/p>\n

This version also contains two typo fixes in the commented example configurations of the gab-sync script (package z-push-kopano-gabsync).
\nIf you have this package installed, you will be alerted at update procedure.
\nYou can maintain the existing config, but should fix these typos manually (see file diff). <\/p>\n

We would like to thank all our community contributors and everyone who tests Z-Push and provides us feedback.<\/p>\n

If you are interested in a detailed overview of the code changes, see the full code diff<\/a><\/p>\n

Install Z-Push 2.3.1 from the final repositories (recommended): https:\/\/wiki.z-hub.io\/display\/ZP\/Installation<\/a>
\nThe tarball is available, the link can be found at ‘Installation’ wiki page.<\/p>\n

Your feedback is highly appreciated.<\/p>\n

Greetings,
\nSebastian
