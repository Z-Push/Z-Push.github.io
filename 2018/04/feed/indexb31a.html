we have just released Z-Push 2.4.1 final (tag 2.4.1 commit db5a7026c70179d169a494afd4ae864c1ff89ce7).

This is a maintenance release with some bugfixes.

There was a change in z-push.conf.php regarding default USE_CUSTOM_REMOTE_IP_HEADER value.
You should upgrade to Z-Push 2.4.1 if you are using fail2ban as Z-Push 2.4.0 might potentially block all requests.

We have changed folder ownership and logrotation for RHEL-based systems. Please check carefully if everything works as expected, especially if you are using nginx.

In the cases when memory_limit was set to -1 in php.ini the sync wouldn’t work. This is now fixed.

Several smaller bugfixes for caldav, carddav (thanks to Bart for contributions!) and imap backends.

Install Z-Push 2.4.1 from the final repositories (recommended) https://wiki.z-hub.io/display/ZP/Installation.

The complete changelog is available here.

Code changes since Z-Push 2.4.0: https://stash.z-hub.io/projects/ZP/repos/z-push/compare/diff?targetBranch=refs%2Ftags%2F2.4.0&sourceBranch=refs%2Ftags%2F2.4.1

Please share your experiences (positive and negative) and give feedback!

Greetings,
Manfred
