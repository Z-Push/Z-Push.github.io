Hello,<\/p>\n

we have just released Z-Push 2.4.3 final<\/strong> (tag 2.4.3<\/strong> commit db9535be7940d0d0c83e1e08de77bd40997398f3<\/strong>).<\/p>\n

This is a maintenance release with some bugfixes and improvements.<\/p>\n

This is the first Z-Push release which supports PHP 7.2 (default PHP version in Ubuntu 18.04). If you are running Ubuntu 18.04 or plan to upgrading to this version, you should upgrade to Z-Push 2.4.3.<\/p>\n

Several smaller issues which were found by coverity (https:\/\/scan.coverity.com\/projects\/z-push<\/a>). Z-Push has now 0.00 defect density!<\/p>\n

We added a new action “listshares” to z-push-admin. It will display all open shares on the system per store. Run
\nz-push-admin -a listshares<\/code>
\nto display open shares for all stores. Run
\nz-push-admin -a listshares -o USER<\/code>
\nto display who opened the shares of this user. Run
\nz-push-admin -a listshares -f FOLDERID<\/code>
\nto display who opened this specific folder.<\/p>\n

Z-Push has a new contributor, Raoul, who fixed a bug in the IMAP backend. Thanks!<\/p>\n

Bart has already made various contributions and in this version he added another improvement to the IMAP backend. <\/p>\n

Also thanks to Matt and Markus for reporting an issue with logrotation on RHEL-based systems and a suggestion how to fix it.<\/p>\n
