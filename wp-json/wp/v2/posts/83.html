Hi,<\/p>\n

today we have released the final version of Z-Push 1.5.4. It is identical to Z-Push 1.5.4RC-705 (only the version and build number were updated).<\/p>\n

Notes:<\/strong>
\nThis is maintenance release for Z-Push 1.5.<\/p>\n

This release is recommended for all Zarafa users which use windows mobiles, Nokia and Android phones.
\nIt is also recommended for the\u00a0 IMAP backend users as it fixes some issues with multipart emails.<\/p>\n

Changes since 1.5.3<\/strong><\/p>\n

General
\n– fixed: Eurosign (\u20ac) gets scrambled when sending email from Windows Mobile (Mantis #456)
\n– fixed: patch to add forgotten licensing exception to LICENSE file (Mantis #462) (thx to Robert Scheck for reporting)
\n– fixed: Streamer and RFC822 class should not be executable by default while z-push-admin should be (Mantis #472) (thx to bencer from Zentyal for reporting)<\/p>\n

Zarafa\/ICS
\n– changed: GAL search returns not all values available in ADS (Mantis #469)
\n– fixed: body truncation not always correct (Mantis #467)
\n– fixed: HTML code in mails sent from Android 2.2 (Mantis #472)<\/p>\n

IMAP backend
\n– fixed: body truncation not always correct (Mantis #467)
\n– fixed: no attachments on the mobile if Content-Disposition header is not set (Mantis #458)
\n– fixed: email is not quoted-printable encoded even if the header states it (Mantis #460)
\n– fixed: email can’t be synchronised with certain date format in header (Mantis #454)<\/p>\n

The version is available at:\u00a0http:\/\/z-push.sourceforge.net\/download<\/a><\/p>\n

Your feedback is highly appreciated.<\/p>\n

Kind regards,
\nManfred
