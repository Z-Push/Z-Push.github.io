Z-Push 2.4.5 final release

by Admin | Nov 7, 2018 | News | 0 comments

Hello,

we have just released Z-Push 2.4.5 final (tag 2.4.5 commit 3b937fb5c79068553198347ffad1f39f2e9d1b51).

This is a maintenance release with some bugfixes and improvements.

Email categories would disappear in Outlook when marking email as read or unread on another client. This issue has been fixed however there might be increased server load as Z-Push now opens the email to load the categories.

We made some changes to Z-Push packaging in order to fix dependencies on Debian 9, logrotate on RHEL 6 and improve performance for installations which use nginx as the webserver.

Using RHEL with Remi PHP 7.x packages and SQL state machine should work now.

Thanks to Nicolas who provided a path to reproduce an issue GAL search bug and a solution how to fix it.

Private items are no longer stripped in own calendar when it’s configured as shared folder in Z-Push config.

Editing recurring tasks on mobile clients now shouldn’t break synchronisation.

Install Z-Push 2.4.4 from the final repositories (recommended) Z-Push Wiki.

The complete changelog is available here.

Code changes since Z-Push 2.4.4: https://stash.z-hub.io/projects/ZP/repos/z-push/compare/diff?targetBranch=refs%2Ftags%2F2.4.4&sourceBranch=refs%2Ftags%2F2.4.5

Please share your experiences (positive and negative) and give feedback!

Greetings,
Manfred
