Hi all,<\/p>\n

today we have released the Z-Push 2.2.3 final version (tag 2.2.3, commit 4cd5b765dfaffda34017354fbcb57e13899ef2c5). This is our first release using git and the new Atlasian stack at https:\/\/jira.z-hub.io<\/a>. Thanks for everybody who was involved in making it happen.<\/p>\n

Important notes<\/strong>
\nThis version contains several bug fixes.
\nS\/MIME signed and encrypted emails are now displayed correctly in Outlook 2013 connected via Z-Push\/ActiveSync for the ZCP backend.
\nMessage move status won’t return error when trying to move a message into trash that cannot be found on the server anymore.
\nWarnings when using autodiscover were fixed.
\nSome enhancements for z-push-admin.<\/p>\n

Changes since Z-Push 2.2.2<\/strong>
\nGeneral
\n[ZP-550] – Warnings in AutoDiscover
\n[ZP-574] – z-push-admin -a fixstates gives errors in z-push.log
\n[ZP-577] – Add legacy function for http_response_code
\n[ZP-621] – LoopDetection code causing full resync after multiple folder changes
\n[ZP-625] – Warnings in index.php<\/p>\n

Zarafa
\n[ZP-593] – S\/Mime signed emails are not displayed correctly in OL13
\n[ZP-609] – Warnings when searching
\n[ZP-624] – Message move should return status OK even if there were errors
\n[ZP-634] – Fix ZP-505 causes all messages to be turned into html as well<\/p>\n

The version is available at: http:\/\/download.z-push.org\/final\/2.2\/<\/a>.<\/p>\n
