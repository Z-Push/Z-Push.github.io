Hello,<\/p>\n

we have just released Z-Push 2.4.2 final<\/strong>** (tag 2.4.2 commit e2a70a79747c8004701f7c915b18587b79f7c59d).<\/p>\n

This is a maintenance release with some bugfixes and improvements.<\/p>\n

We have improved Z-Push packages. For debian-based systems Felix changed the dependency handling.
\nRHEL-based systems now should have the correct permissions and ownership for state and log folders. Please check carefully if everything works as expected.<\/p>\n

In certain cases Z-Push triggers folder re-sync if the client requests deletion a lot of items. Sometimes this a desired behaviour and the folder re-sync is not necessary. Michele added new feature to take a threshold into account. Check [Z-Push Wiki](https:\/\/wiki.z-hub.io\/display\/ZP\/Hidden+configuration+parameters) for more information.<\/p>\n

For the Kopano backend we fixed saving picture for a contact created on mobile and unsetting properties for tasks.<\/p>\n

Niels added authentication with GSSAPI support for IMAP servers. Therefore changes in the config of the IMAP backend might be necessary.<\/p>\n

Bart fixed several bugs for imap and caldav backends.<\/p>\n

Thank you all for the contributions!<\/p>\n

Install Z-Push 2.4.2 from the final repositories (recommended) https:\/\/wiki.z-hub.io\/display\/ZP\/Installation<\/a>.<\/p>\n

The complete changelog is available here<\/a>.<\/p>\n
