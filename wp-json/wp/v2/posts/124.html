Hi all,<\/p>\n

Today we have released the Z-Push 2.1.0 final version (build 1750).
\nIt contains several new features like S\/Mime support and administration features. Windows 8 ActiveSync is now also supported.
\nCompared to 2.1.0 Milestone 2 this version has various general fixes, regarding S\/Mime, Blackberry 10 and calendaring.<\/p>\n

For better performance of Windows 8 mail clients changes in the state files are necessary.
\nWhen you upgrade from Z-Push 2.0.x or 2.1.0 M1 you need to migrate your Z-Push states – see instructions below.<\/p>\n

Please report us your experience, about this release in general and please also about the upgrade process.<\/p>\n

Important – NOTE<\/strong>
\nThis version of Z-Push only works with ZCP 7.0.6 or later.<\/p>\n

Important – UPGRADE from Z-Push 2.0.x<\/strong>
\nThe states of Z-Push 2.0.x and 2.1.0 M1 are not compatible, but a state migration script is available in the tools folder.<\/p>\n

To upgrade, please follow these steps:
\n1. Upgrade your z-push files to the new version
\n2. Restart apache\u00a0 (important or your upgrade might fail!)
\n3. Mobiles which connect will receive a temporary synchronization error (only visible if you explicitly click “refresh” on the mobiles)
\n4. Run the upgrade script located in the tools folder. The files distributed by us contain a correct configuration of this script to access the config file and locate the state files. If your file locations are different, please check the path settings in the top of the migration script.
\n5. After running the migration script, mobiles will continue synchronizing without any re-synchronization<\/p>\n

You can run the script multiple times, but normally there is no need to do so.<\/p>\n

Changes compared to Z-Push 2.0.x:<\/strong><\/p>\n

New features:
\n[ZP-36] – Feature to resynchronize a single folder or folder type of a user
\n[ZP-43] – Implement ValidateCert command
\n[ZP-44] – Implement ResolveRecipients command
\n[ZP-45] – Implement S\/Mime
\n[ZP-237] – Show if folders are fully synchronized in z-push-admin device details
\n[ZP-272] – Command MoveItems should check if messages are in sync interval
\n[ZP-328] – Proxied Z-Push could show remote IP instead of proxy IP
\n[ZP-379] – Reply\/Forward flags are not synced from the server to the mobile<\/p>\n

Full change log is available\u00a0here<\/a>.<\/p>\n

The version is available at:\u00a0http:\/\/zarafa-deutschland.de\/z-push-download\/final\/2.1\/<\/a>.<\/p>\n

Your feedback is highly appreciated.<\/p>\n

Kind regards,
\nManfred
