Today we have released Z-Push version 1.5.5 final.<\/p>\n

Notes:
\nThis is maintenance release for Z-Push 1.5.<\/p>\n

This release is recommended for all Zarafa users, especially when using ZCP 7.0 or having a lot of Z-Push users.
\nIt also fixes email addresses’ and birthdays’ problem for contacts created on an android.
\nIt is also recommended for the IMAP backend users as it fixes some issues regarding special characters in folders.<\/p>\n

Changes since 1.5.4<\/strong><\/p>\n

General
\n– fixed: typo prevents LOOSE_PROVISIONING (Mantis #476) (thx to \u2018settel\u2019 for reporting)
\n– fixed: Synchronizing an empty folder an additional ICS exporter is registered on each request (Mantis #480)
\n– fixed: Moxier Tasks 1.1.1 on iOS presents errors (Mantis #490)
\n– fixed: Contacts Birthday not properly saved from Android (Mantis #487)
\n– fixed: No conflict detected for deleted or read flag changed messages (Mantis #489)<\/p>\n

Zarafa\/ICS
\n– fixed: Lazy conflict detection shows warnings and does not check conflicts (Mantis #481)
\n– fixed: Appointment is not synchronized after meeting invitation was accepted in the Webaccess of ZCP 7 (Mantis #475)
\n– fixed: Saving empty categories fails (Mantis #468)
\n– fixed: GAL search shows ‘undefined offset of PR_ACCOUNT’ warnings (Mantis #488)
\n– fixed: Meeting request responses can not be processed correctly (Mantis #485)
\n– fixed: Contacts created on Android phones present doubled names (Mantis #486)
\n– fixed: Changing a meeting request to all day event shows warnings (Mantis #491)<\/p>\n

IMAP backend
\n– fixed: Folders with special chars break synchronisation (Mantis #463)<\/p>\n

The version is available at:\u00a0http:\/\/z-push.sourceforge.net\/download<\/a><\/p>\n

Your feedback is highly appreciated.<\/p>\n

Kind regards,
\nManfred
