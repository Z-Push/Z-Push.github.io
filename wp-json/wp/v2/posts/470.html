Hi all,<\/p>\n

we have just released Z-Push 2.3.3 final (tag 2.3.3, commit d5759a2498e2ef9dfd3b8c5b958f88654ec71a3a).<\/p>\n

This version contains fixes related to Autodiscover (high characters in passwords, fail2ban support) and for the Kopano OL Extension (KOE).<\/p>\n

Until this version, opening a shared calendar also gives you full access to private calendar items of the owner.
\nThis is fixed in this version but in order to remove the private entries, the folder needs to be resynchronized.
\nThis can be realized by one of the following ways:
\n1. Best option: Use a provided script to list all users with shared calendars and resynchronize them from the server.
\nThe script is available here: https:\/\/stash.z-hub.io\/projects\/ZP\/repos\/z-push\/browse\/tools\/list-shared-folders.php<\/a>
\nUsage instructions can be found within the file (line 50).
\n2. Easy but with overhead: Resynchronize all calendars using “z-push-admin -a resync -t calendar”. This will cause all calendars from all users to be resynchronized.
\nThis could be a lot of overhead and produce high load on your server (for many users\/calendars). Prefer option 1!
\n3. Manual fix: Ask the users to close and reopen the shared calendar in Outlook.<\/p>\n

We would like to thank all our community contributors and everyone who tests Z-Push and provides us feedback. Please continue doing that.<\/p>\n

The full changelog is available here<\/a>.<\/p>\n
