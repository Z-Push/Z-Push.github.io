2015-10-19: We had to take this version offline as it introduced a bug in all backends (ZP-723). We will release a Z-Push 2.2.5 which will include a fix for it and changes from 2.2.4 today.

today we have released the Z-Push 2.2.4 final version (tag 2.2.4, commit fecec66ca6a266e526efe7304e2823e822d31882). We had to remove ZP-672 from the final release because it has caused problems with GAL search.

Important notes
This version contains quite some bug fixes and improvements, most of them being the contributions from the community. Thanks a lot to Francisco, Ralf and Vincent.
Outlook with ActiveSync was working slow or not at all in certain cases due to Global WindowsSize not being honoured.
Adjustments to make Z-Push work with PHP-Composer.
Functionality to auto load backend was added.
There were improvements for the DiffBackend performance.
Z-Push logs and legal message now display the git commit information.

The full changelog is available here.

The version is available at: http://download.z-push.org/final/2.2/.

Your feedback is highly appreciated.

Kind regards,
Manfred
Z-Push dev team

]]> https://z-push.org/z-push-2-2-4-final-released/feed/ 0 Z-Push 2.2.3 final released https://z-push.org/z-push-2-2-3-final-released/ https://z-push.org/z-push-2-2-3-final-released/#respond Wed, 02 Sep 2015 23:36:53 +0000 http://z-push.org/?p=339 Hi all,

today we have released the Z-Push 2.2.3 final version (tag 2.2.3, commit 4cd5b765dfaffda34017354fbcb57e13899ef2c5). This is our first release using git and the new Atlasian stack at https://jira.z-hub.io. Thanks for everybody who was involved in making it happen.

Important notes
This version contains several bug fixes.
S/MIME signed and encrypted emails are now displayed correctly in Outlook 2013 connected via Z-Push/ActiveSync for the ZCP backend.
Message move status won’t return error when trying to move a message into trash that cannot be found on the server anymore.
Warnings when using autodiscover were fixed.
Some enhancements for z-push-admin.

Changes since Z-Push 2.2.2
General
[ZP-550] – Warnings in AutoDiscover
[ZP-574] – z-push-admin -a fixstates gives errors in z-push.log
[ZP-577] – Add legacy function for http_response_code
[ZP-621] – LoopDetection code causing full resync after multiple folder changes
[ZP-625] – Warnings in index.php

Zarafa
[ZP-593] – S/Mime signed emails are not displayed correctly in OL13
[ZP-609] – Warnings when searching
[ZP-624] – Message move should return status OK even if there were errors
[ZP-634] – Fix ZP-505 causes all messages to be turned into html as well

The version is available at: http://download.z-push.org/final/2.2/.

Your feedback is highly appreciated.

Kind regards,
Manfred
Z-Push dev team
